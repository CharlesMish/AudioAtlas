# AudioAtlas JSON contracts

This document describes the public alpha JSON artifacts generated by
AudioAtlas `0.2.0a4`.

| Artifact | Schema version | Purpose |
|---|---:|---|
| `summary.json` | `0.2.1` | Canonical measurements, identity assertion, and run provenance for one analyzed range. |
| `findings.json` | `0.2.0` | Rule-based checks derived from the summary. |
| `catalog_summary.json` | `0.2.0` | Neutral folder-level index of successful and skipped files. |
| `revision_diff.json` | `0.1.0` | Guarded descriptive deltas between two asserted revisions of one track. |
| calibration replay JSON | `0.1.0` | Anonymous prompt churn against a hash-verified review ledger. |

Schema constants live in `src/audioatlas/release.py`.

## Privacy contract

By default, generated JSON does not contain resolved source or output paths.

Single-track metadata contains:

```jsonc
{
  "path": "song.wav",
  "filename": "song.wav",
  "path_kind": "basename",
  "local_paths_included": false
}
```

Catalog folders use the same basename policy. `--include-local-paths` opts into
absolute resolved paths and sets `path_kind` to `absolute` and
`local_paths_included` to `true`.

Decoder exception text is sanitized before it reaches user-facing output.

When `--track-id` is used, the raw token is not serialized. `source_identity`
contains only its SHA-256 digest. Matching digests prove that the same token was
supplied; they are not audio fingerprints or independent track recognition.

## `summary.json`

Top-level shape:

```jsonc
{
  "schema_version": "0.2.1",
  "metadata": {},
  "source_identity": {},
  "analysis_config": {},
  "analysis_provenance": {},
  "levels": {},
  "rms_envelope": {},
  "crest_factor_timeline": {},
  "peak_timeline": {},
  "average_spectrum": {},
  "spectral_shape": {},
  "band_power_timeline": {},
  "band_energy_timeline": {},
  "onset_density": {},
  "chroma_cqt": {},
  "short_term_lufs": {},
  "stereo_correlation": {},
  "mid_side_energy": {},
  "plots": [],
  "graphs": {}
}
```

`band_energy_timeline` is a deprecated compatibility alias containing the same
object as canonical `band_power_timeline` during the 0.2 alpha line.

### `metadata`

| Field | Type | Meaning |
|---|---|---|
| `path` | string | Basename by default; resolved path only by explicit opt-in. |
| `filename` | string | Source basename. |
| `path_kind` | `basename` or `absolute` | Disclosure mode. |
| `local_paths_included` | boolean | Whether local absolute paths were requested. |
| `samplerate` | integer | Decoded sample rate in Hz. |
| `channels` | integer | Decoded channel count. |
| `frames` | integer | Frames in the analyzed range. |
| `duration_seconds` | number | Duration of the analyzed range. |
| `format`, `subtype`, `endian` | string or null | Decoder metadata. |
| `source_start_seconds`, `source_end_seconds` | number | Range in the source file. |
| `source_duration_seconds` | number | Full decoded source duration. |


### `source_identity`

```jsonc
{
  "kind": "user-supplied-sha256",
  "track_id_sha256": "64 lowercase hexadecimal characters"
}
```

Without `--track-id`, `kind` is `none` and `track_id_sha256` is null. The
identity is intentionally an opaque user assertion suitable for grouping
exports of one track. The token should be long and non-obvious when its digest
will be shared. AudioAtlas omits it from output, but a shell may retain command
arguments in local history.

### `analysis_provenance`

The provenance block is path-safe and records:

| Field | Meaning |
|---|---|
| `format_version` | Provenance-object format version. |
| `audioatlas_version`, `summary_schema_version`, `finding_ruleset_version` | Release identities at generation time. |
| `analysis_config_sha256` | Canonical SHA-256 of the complete serialized `AnalysisConfig`. |
| `measurement_code_sha256` | Hash of the measurement, loader, and utility source files used by the installed package. Finding/report presentation code is excluded. |
| `finding_rule_code_sha256` | Separate hash of the finding implementation. |
| `dependencies` | Installed versions of NumPy, SciPy, SoundFile, Librosa, Numba, and pyloudnorm. |
| `decoder` | SoundFile and libsndfile versions. |
| `measurement_methods` | Named implementation details, including approximate-true-peak oversampling and the non-standards-grade caveat. |
| `environment` | Python implementation/version, operating system, machine class, and byte order. |
| `compatible_analysis_sha256` | Hash of configuration, measurement code/methods, dependencies, and decoder. A match permits a comparison with an environment caveat. |
| `exact_environment_sha256` | Hash of the compatible payload plus the environment block. A match is the strongest comparability level AudioAtlas currently records. |

These signatures improve reproducibility and comparison honesty. They do not
guarantee bit-identical floating-point results, validate a standards-grade true
peak meter, or establish that two source files are revisions of the same music.

### `levels`

Contains global and per-channel sample peak, approximate true peak, RMS, crest
factor, integrated LUFS, PLR, clipping counts, DC offset, duration, and
warnings. Per-channel fields follow input channel order.

PLR is `true_peak_dbtp - integrated_lufs`. A constant loudness-normalization
gain changes both operands equally and does not change PLR.

### Relative spectral blocks

`average_spectrum` contains:

```jsonc
{
  "band_measurement": "relative_mean_power_per_fft_bin",
  "band_mean_power": {
    "bass": {
      "low_hz": 60.0,
      "high_hz": 120.0,
      "mean_power_db": -3.2,
      "energy_db": -3.2
    }
  },
  "highest_mean_power_band": "bass",
  "band_energies": {},
  "strongest_band": "bass"
}
```

`band_mean_power` and `highest_mean_power_band` are canonical. `band_energies`,
`energy_db`, and `strongest_band` are deprecated aliases.

The values are mean spectral power per included FFT bin, normalized within the
analysis view. They are **not** total energy integrated across differently
sized frequency bands and are not calibrated dBFS.

`band_power_timeline` contains:

- `measurement: "relative_mean_power_per_fft_bin"`;
- timing/configuration fields;
- per-band median, mean, min, max, relative thresholds, and time ranges;
- canonical `highest_mean_power_band_by_median`;
- deprecated `strongest_band_by_median`;
- structured warnings.

### Other analysis blocks

- `rms_envelope`: frame RMS timeline summary.
- `crest_factor_timeline`: frame sample-peak-to-RMS contrast.
- `peak_timeline`: per-frame peak and clipping/near-clipping ranges.
- `spectral_shape`: centroid, 85%/95% rolloff, bandwidth, and ranges.
- `onset_density`: onset-strength activity summary and ranges; not hit count or
  punch quality. `mel_bands` records the filter count used by the onset
  calculation; it stays at 128 for normal FFT sizes and is reduced for very
  small FFTs to avoid empty filters.
- `chroma_cqt`: pitch-class energy; not key or chord detection.
- `short_term_lufs`: blockwise short-term loudness or a short-file warning.
- `stereo_correlation`: valid-frame L/R relationship and low-correlation
  ranges.
- `mid_side_energy`: mid/side levels and side-to-mid context.

### `plots` and `graphs`

`plots` lists selected stable PNG filenames in render order. `graphs` records:

| Field | Type | Meaning |
|---|---|---|
| `profile` | string | `minimal`, `standard`, or `full`. |
| `selected` | list[string] | Selected graph keys in registry order. |
| `available` | list[string] | All registered graph keys. |
| `selected_filenames` | list[string] | Published PNG filenames. |

Graph selection does not reduce the analysis blocks serialized above.

## `findings.json`

Top-level shape:

```jsonc
{
  "schema_version": "0.2.0",
  "ruleset_version": "0.2.0a2",
  "count": 1,
  "all_count": 1,
  "max_findings": 8,
  "findings_suppressed_count": 0,
  "findings": [],
  "findings_shown": [],
  "all_findings": []
}
```

`findings` and `findings_shown` are equivalent display lists retained for
compatibility. `all_findings` preserves the full sorted set before the report
cap.

Each finding contains:

| Field | Type | Meaning |
|---|---|---|
| `rule_id` | string | Stable machine identity. |
| `rule_version` | integer | Version of trigger/meaning for that rule. |
| `severity` | `info`, `warning`, or `issue` | Review priority, not a quality score. |
| `category` | string | Measurement area. |
| `title` | string | Human label. |
| `measured_value`, `threshold`, `unit` | number/null, number/null, string | Optional simple primary trigger. Null for grouped multi-metric rules. |
| `evidence` | string | Human-readable evidence. |
| `evidence_items` | list[object] | Typed metric path, label, value, unit, comparator, threshold, and ranges. |
| `why_it_matters` | string | Narrow supported significance. |
| `does_not_mean` | string | Explicit interpretation boundary. |
| `suggested_checks` | list[string] | Human checks, not automatic fixes. |
| `time_ranges` | list[object] | Display-relevant ranges in seconds. |
| `associated_graphs` | list[string] | Stable graph keys. |
| `confidence` | string | Confidence in rule application. |

See `docs/FINDING_RULES.md` for enabled rules and calibration requirements.

## `catalog_summary.json`

Top-level fields:

| Field | Type | Meaning |
|---|---|---|
| `schema_version` | string | `0.2.0`. |
| `generated_at` | string | UTC ISO-8601 timestamp. |
| `input_folder`, `output_folder` | string | Basenames by default; absolute only by opt-in. |
| `path_kind`, `local_paths_included` | string, boolean | Disclosure mode. |
| `track_count` | integer | Successfully analyzed files. |
| `skipped_files` | list[object] | Filename, reason, and optional status such as `analysis_failed`. |
| `tracks` | list[object] | Compact per-track records and relative report links. |
| `statistics` | object | Count/min/median/max/mean/missing by metric. |
| `common_patterns` | list[object] | Caveated traits shared by more than 60% of successful tracks. |
| `decoded_audio_context` | object | Lossy-source context. |

Canonical per-track broad-band field is `highest_mean_power_band`;
`strongest_band` remains a deprecated alias. Catalogs do not rank tracks.
Unreadable supported-extension files appear in `skipped_files`; successful
tracks remain available unless strict batch mode was requested.

## Compatibility policy

During the 0.2 alpha line, renamed band-power fields retain explicit aliases.
New consumers should prefer canonical names.

Bump a schema version when a documented field is removed, renamed without an
alias, retyped, or changes meaning. Adding an optional field is normally
additive. Every schema change must update tests, this document, the
[compatibility policy](COMPATIBILITY.md), and the changelog in the same patch.

## `revision_diff.json`

The diff command accepts completed report folders (or their `summary.json`
files) and writes one static revision-delta bundle. Its JSON contains:

- `comparison_kind: "same-track-revision"`;
- labels and summary/finding snapshot hashes for A and B;
- a `same_track` assessment (`matched` hashed token or explicit user assertion);
- a `comparability` assessment (`exact`, `compatible`, `unknown`, or
  `incomparable`) and whether an override was used;
- source-scope caveats for differing sample rate, channels, range start, or
  duration;
- scalar `metric_deltas` and broad-band median deltas, always `B - A`;
- finding `appeared`, `disappeared`, `changed`, and `unchanged_count`;
- an interpretation boundary prohibiting ranking, recommendation, causation,
  and cross-track use.

The output directory must be separate from both source report directories.
Conflicting non-null track-identity digests are a hard error. Missing identity
requires `--confirm-same-track`. Unknown or incompatible provenance is refused
by default and can only be emitted with `--allow-incomparable`, which remains
recorded prominently in JSON, Markdown, and HTML.

## Calibration replay JSON

`replay_calibration_rules.py` consumes the private asset map and anonymous
human-review ledger created by `prepare_calibration_review.py`. Before replay,
it recomputes the reviewed `summary.json`/`findings.json`/manifest digest for
each asset and refuses changed evidence. It then regenerates findings from the
saved summary with the current checkout and emits anonymous per-rule statuses:
`appeared`, `disappeared`, `changed`, or `unchanged`.

The public replay output contains asset IDs, hashes, baseline/candidate
finding-rule implementation fingerprints, rule identities/versions, visibility,
changed fields, and ruleset versions. It omits filenames, audio
paths, report paths, and audio content. It replays finding logic only; it does
not reopen audio, re-run measurements, or replace human listening labels.
